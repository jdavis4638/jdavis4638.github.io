About Me

Favorite Past Project

Though not a programming project, I thoroughly enjoyed the work in the cyber-security class offered at CU Boulder.

The projects in that class largely centered around a "Capture the Flag" style, wherein we were tasked with breaking into faux-business websites. The intrigue was due to the amount of effort the instructor and TAs put into the class in order to make the process feel as immersive as possible.

Examples:

  • "Hacking" into a business using SQL injections.
  • Using tools like wireshark to search for unencrypted web traffic to and from a specific IP adress.
  • Sending problem packets to learn about where faults in the web traffic may exist, and using these faults to gain access to administrator passwords.
  • Using these admin passwords to gain access to systems such as security cameras where we could find passwords on sticky-notes at employee desks, and using these to do fun things in the system.

Final Project

Image of a dextrously impaired man in a wheelchair.

The final project was the one in which I had to relearn a lot of what I learned during this semsester. This required an actual understanding of the different concepts described in the other projects while also requiring good structure.

For example: while we have worked with grids and flex boxes in this class before, I had to learn how to put these things together in a more complicated layout. It wasn't so much about remembering the code, it was more about understanding it. It was also useful for me to have to go back to the earlier course modules for the content of this project; it acted as a refresher for all that we learned during the course.

There are some things I would change about it if I had the option to:

  • Having a functioning table of contents. Currently, the table of contents dropdown does not work, and there's an issue with a key aspect of accessible design in the current implementation. Namely, it functions completely off of mouse control, requiring the user to mouseover the dropdown in order to click to go to different sections of a page.
  • I did not add context specific text to each of the images for the visually impaired.
  • I'm still a little unsure on the actual semantic structure of the page. This comes back to the idea of "understanding", where I'll admit that my semantic structure is based on remembering how to code html, rather than understanding it. This is by far my biggest takeaway from this project, and something I will be working on in the future.
